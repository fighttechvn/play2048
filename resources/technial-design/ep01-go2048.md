# EP01 Technical Design: go2048 Game

## Technologies
- TypeScript + [Vite](https://vite.dev) bundler.
- [PixiJS v7](https://pixijs.com) (WebGL 1) for rendering.
- Capacitor 6 for native packaging (`@capacitor/preferences`, `haptics`,
  `splash-screen`, `status-bar`).

## Entry Points
- `src/main.ts` — bootstrap (PixiJS `Application`, scene, input, splash).
- `src/game/gameScene.ts` — view/controller (scene graph, animations, HUD).

## Architecture (layered, no framework)
```
input → GameScene (PixiJS view) → Board (pure logic) → storage (Preferences)
```
The game logic is decoupled from rendering: `board.ts` is pure TypeScript and
returns plain data; `GameScene` turns that data into animations.

## Flow
1. Player swipes / presses a key → `input.ts` emits a `Direction`.
2. `GameScene.handleMove(dir)` calls `board.move(dir)`.
3. `board.move` (pure) returns `{ movements, merges, spawn, over }`.
4. `Tweener` animates tiles to their target cells, then pops merges/spawns.
5. `GameScene.refreshHud()` updates score/best; `storage.saveState()` persists.
6. Win / over states open an overlay.

## Entities
- `Tile { id, value }`
- `BoardState { grid, nextId, score, best, won, keptPlaying }`
- `Direction = 'up' | 'down' | 'left' | 'right'`
- `Movement { id, to, mergedAway }`, `Merge { id, value, at }`
- Theme `Palette` (Dark/Light), `Locale` (en/vi/ko/zh/ar)

## Modules
- `board.ts` — pure 2048 engine (move/merge/spawn/win/over, serialize/load).
- `gameScene.ts` — scene graph, animations, HUD, theme/locale controls.
- `tileView.ts` — one tile (rounded rect + number).
- `theme.ts`, `i18n.ts`, `storage.ts`, `input.ts`, `tween.ts`.

## Tests
- Unit: `board.ts` move/merge/spawn/win/over + serialize round-trip.
- Integration: a scripted sequence of moves reaches a known board/score.
- Screenshot e2e: capture light/dark/2048 states (see `e2e.sh`).
