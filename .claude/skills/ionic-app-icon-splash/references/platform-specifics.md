# Platform Specifics

## iOS icon

- **Source must NOT have transparency.** iOS rejects icons with alpha channels — App Review rejects the build.
- **Source must NOT have rounded corners.** iOS applies its own squircle mask. Pre-rounded corners get visible as a darker ring.
- The current Apple guideline is **1024×1024 PNG, sRGB or Display P3, flattened**.

## iOS splash

- iOS uses a single PNG scaled per device. The system itself uses a launch storyboard that points to the generated `Splash.imageset/`.
- For dark-mode support, supply `splash-dark.png` and `capacitor-assets` writes both into the imageset with the right trait collection.

## Android adaptive icon

Android 8+ uses adaptive icons composed of two layers:

- **Foreground**: your logo / mark, must fit inside a ~66% safe zone (the launcher can apply round / squircle / teardrop masks).
- **Background**: a solid color or simple gradient that fills behind the foreground.

If you only provide `icon-only.png`, `capacitor-assets` falls back to a non-adaptive icon — works, but looks dated on modern Android.

## Android splash

Android uses a `windowSplashScreen` (Android 12+) and a legacy `Splash` activity for older versions. `capacitor-assets` configures both. The `@capacitor/splash-screen` plugin then controls when to hide it — typically right after the app finishes initial bootstrap.

```typescript
// after first useful paint
import { SplashScreen } from '@capacitor/splash-screen';
await SplashScreen.hide();
```

If `SplashScreen.hide()` is never called, the splash sticks until you navigate. In `capacitor.config.ts`:

```typescript
{
  // ...
  plugins: {
    SplashScreen: {
      launchShowDuration: 2000,
      launchAutoHide: true,
      backgroundColor: '#000000',
      // Note: `androidSplashResourceName` and most legacy splash options below
      // apply to Android 11 and earlier. Android 12+ uses `windowSplashScreen`,
      // configured via `colors.xml` / `styles.xml` generated by @capacitor/assets.
      androidSplashResourceName: 'splash',
      androidScaleType: 'CENTER_CROP',
      showSpinner: false,
      splashFullScreen: true,
      splashImmersive: true,
    },
  },
}
```

`launchAutoHide: true` removes the need to call `hide()` manually — set this for the simplest case.

## Status bar

While the splash is showing, the status bar should be hidden / transparent to avoid a visible bar over the splash. Use `@capacitor/status-bar`:

```typescript
import { StatusBar, Style } from '@capacitor/status-bar';

// Style.Dark = dark text/icons (use when the splash background is LIGHT)
// Style.Light = light text/icons (use when the splash background is DARK)
// Counter-intuitive — the style refers to the foreground content, not the
// background.
await StatusBar.setStyle({ style: Style.Dark });
```

## Verify

After generation:

```bash
npx cap sync
npx cap open ios          # check Assets.xcassets / AppIcon.appiconset is populated
npx cap open android      # check res/mipmap-* / drawable* are populated
ionic serve               # browser doesn't show native icon, but splash CSS does render
```

For a real visual check, build to a device:

```bash
npx cap run ios --target=<device>
npx cap run android --target=<device>
```
